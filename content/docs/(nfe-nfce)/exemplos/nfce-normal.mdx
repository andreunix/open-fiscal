## Contexto fiscal

Venda **presencial** no varejo a consumidor final (`mod=65`, `indFinal=1`, `indPres=1`). O destinatário pode ser **omitido** dentro dos limites estaduais; o pagamento é obrigatório e o **QR Code** vai em `infNFeSupl`, fora de `infNFe`. Vários grupos de transporte e tributos do modelo 55 são **proibidos** aqui.

> ⚠️ **Valores fictícios, trecho ilustrativo.** Chave, CNPJ, valores e o `qrCode` são inventados. Gere o QR Code conforme o [manual vigente](/docs/danfe/danfe-nfce-qrcode); confirme nomes-folha e cardinalidade no [schema vigente](/docs/schemas).

## Campos críticos

| Campo | Valor | Por quê |
|---|---|---|
| `mod` / `tpImp` | `65` / `4` | NFC-e usa DANFE NFC-e — ver [DANFE NFC-e e QR Code](/docs/danfe/danfe-nfce-qrcode) |
| `indFinal` / `indPres` | `1` / `1` | consumidor final, presencial |
| `CRT` × grupo ICMS | `1` → `ICMSSN102` (CSOSN) | Simples usa **CSOSN** — ver [Tributos](/docs/leiaute-e-rejeicoes/tributos) |
| `tPag` | `17` (PIX) | tabela do MOC/NT; pagamento obrigatório — ver [Grupos finais](/docs/leiaute-e-rejeicoes/grupos-finais#pagamentos) |
| `infNFeSupl/qrCode` | versão vigente | fica **fora** de `infNFe`; gere pelo CSC — ver [QR Code](/docs/danfe/danfe-nfce-qrcode) |

## Validações esperadas

- Sucesso: **`cStat 100`**. O QR Code deve resolver para a consulta da chave; o DANFE NFC-e é impresso/exibido.
- Grupos **proibidos** no modelo 65 (transporte detalhado, IPI, II, partilha, ICMSUFDest, devolução de tributos) não devem ser serializados — ver [Tributos](/docs/leiaute-e-rejeicoes/tributos) e [Totais](/docs/leiaute-e-rejeicoes/totais-e-fechamento).

## Rejeições comuns

| `cStat` | Causa | Onde corrigir |
|---|---|---|
| 781 | emissor não habilitado para NFC-e (substituiu a denegação) | [Identificação](/docs/leiaute-e-rejeicoes/identificacao-e-atores) |
| 725 | CFOP não permitido na NFC-e (`I08-150`) | [Tributos](/docs/leiaute-e-rejeicoes/tributos) |
| 387/388/grupo proibido | grupo do modelo 55 indevido no 65 | remover o grupo |
| 297/298 | assinatura ou QR Code fora do padrão | [Erros de assinatura](/docs/seguranca/erros-comuns) |

## Montando o caso completo
