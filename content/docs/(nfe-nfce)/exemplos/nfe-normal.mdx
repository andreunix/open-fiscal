## Contexto fiscal

Venda de mercadoria de um contribuinte do **regime normal** (`CRT=3`) para outro contribuinte na **mesma UF** (`idDest=1`), tributada integralmente pelo ICMS (`CST=00`). É o caso-base do modelo 55: serve de esqueleto para os demais cenários, que mudam finalidade, destino ou grupos específicos.

> ⚠️ **Valores fictícios, trecho ilustrativo.** Chave, CNPJ, valores e datas são inventados. Confirme nomes-folha, ordem e cardinalidade no [schema vigente](/docs/schemas), recalcule o DV da chave e os totais. Não use como fixture pronta.

## Campos críticos

| Campo | Valor | Por quê |
|---|---|---|
| `mod` / `tpAmb` / `tpEmis` | `55` / `2` / `1` | definem documento, ambiente e emissão normal — ver [Identificação](/docs/leiaute-e-rejeicoes/identificacao-e-atores) |
| `CRT` × grupo de ICMS | `3` → `ICMS00` (CST) | regime normal usa **CST**, não CSOSN — ver [Tributos](/docs/leiaute-e-rejeicoes/tributos#escolha-do-grupo-de-icms) |
| `CFOP` × `idDest` | `5102` × `1` | CFOP 5.xxx é operação **interna**; precisa casar com o destino |
| `indTot` | `1` | `vProd` do item compõe o total — ver [Itens](/docs/leiaute-e-rejeicoes/itens-e-produtos) |
| `vICMS` | `vBC × pICMS` | `1000,00 × 18% = 180,00`; confere com `ICMSTot` |
| `vNF` | `1000.00` | fórmula `W16-10` — ver [Totais](/docs/leiaute-e-rejeicoes/totais-e-fechamento) |

## Validações esperadas

- XSD válido → assinatura XMLDSig sobre o `Id` de `infNFe` → autorização. Ver [pipeline](/docs/leiaute-e-rejeicoes/pipeline-de-validacao).
- Sucesso: **`cStat 100`** (Autorizado o uso da NF-e). Guarde o `protNFe` junto ao XML autorizado.

## Montando o caso completo

Este esqueleto recebe os grupos dos demais cenários: para varejo presencial use [NFC-e normal](/docs/exemplos/nfce-normal); para finalidades diferentes, [devolução](/docs/exemplos/nfe-devolucao), [complementar](/docs/exemplos/nfe-complementar) ou [ajuste](/docs/exemplos/nfe-ajuste).
