A **NT 2016.002** é o marco de publicação do **leiaute versão 4.00** da NF-e e NFC-e ("Versão Nacional 2016"). Ela consolida em um único schema as mudanças introduzidas pelas NTs 2015.002 (QR Code, formas de pagamento, regras de validação) e 2015.003 (ICMS interestadual, CEST), além de introduzir mudanças próprias de protocolo, Web Services e leiaute.

> 🕒 Histórico: v1.61 (setembro de 2018). Leiaute v4.00 em produção. A v1.61 contém apenas correções de documentação e schema sem impacto operacional para as empresas.

## O que muda

| Tema | Detalhe |
|---|---|
| **Leiaute v4.00** | Versão definitiva do schema; substitui v3.10. Campos consolidados: CEST, grupo NA (ICMS UF destino), grupo YA (formas de pagamento com `tpIntegra`), grupo ZX (`qrCode`) e grupos rastreabilidade (I80) e medicamentos/matérias-primas (K) |
| **Protocolo TLS** | Migração de SSL para TLS nas conexões com os Web Services |
| **SOAP/WSDL** | Nome dos Web Services e parâmetros de entrada/saída padronizados; variáveis no SOAP Header definidas formalmente |
| **Rastreabilidade (I80)** | Novo grupo para informar lote de fabricação, validade, número de série e código de fabricante de produto rastreável |
| **Medicamentos e matérias-primas (K)** | Detalhe específico de ANVISA/lote/validade/data de fabricação/autorização de isenção de registro |
| **ICMS Tributação=00** | Novos campos `vBCFCPST`, `vFCPST`, `vFCPSTRet` para FCP retido anteriormente |
| **Grupo de Repasse ICMS ST** | Grupo para repasse de ICMS-ST retido anteriormente em operação interestadual |
| **Campos DANFE** | Seção 5 da NT traz campos adicionais de representação obrigatória no DANFE |

## Implicação de implementação

> **Implementação:** o schema vigente é o v4.00. Não aceite ou gere NF-e em schema v3.10 — o autorizador rejeita desde 02/08/2018. Valide todos os grupos novos (NA, I80, K, ZX) conforme as regras de validação desta NT.
