Esta seção tem **uma página por Nota Técnica posterior ao MOC 7.0**. Cada página é o *resumo navegável por NT* (changelog): descreve o que aquela NT introduz — campos, regras de validação, cronograma — e **aponta** para a página de tema onde cada delta já foi dobrado.

> 🔄 As NTs vivem em **duas camadas**. Quem lê por **tema** usa o overlay dobrado na página de leiaute/evento; quem lê por **Nota Técnica** usa a página à parte desta seção. A página à parte **não duplica** o overlay — linka para ele.

## Duas camadas

| Camada | O que é | Onde aparece |
|---|---|---|
| **A — overlay sobre o MOC** | o delta da NT dobrado **em contexto**, ao lado do grupo XML ou do `cStat` a que pertence | seção `## Overlay de NTs` das páginas de tema |
| **B — página à parte** | o *changelog por NT*: o que muda, campos, cronograma, com link para o overlay | esta seção `notas-tecnicas/` |

## Contagem

As **39 Notas Técnicas posteriores ao MOC 7.0** — as únicas **não consolidadas no MOC** — se dividem em:

| Grupo | Quantidade |
|---|---:|
| 🔄 Revisões vigentes (uma página à parte cada) | **36** |
| 🕒 Revisões superadas (sem página à parte) | **3** |

> As superadas — **NT 2016.003 v3.62** (→ v3.70) e **NT 2025.002 v1.36/v1.40** (→ v1.50) — não ganham página. As 65 NTs compiladas no MOC 7.0 também não entram aqui; o catálogo completo está em [Notas Técnicas no MOC 7.0](/docs/referencia/notas-tecnicas).

## Camadas para o open-fiscal

```txt
MOC 7.0 + Anexo I       = base até nov/2020 (65 NTs compiladas)
Overlays pós-MOC        = camada atual incorporada (36 NTs vigentes)
Schemas atuais          = fonte prática de validação XML
Tabelas vigentes        = fonte prática de códigos/domínios
```

> A data exibida no índice mestre é a da revisão vigente. Uma NT antiga (ex.: 2014.001, 2015.001) reaparece como overlay porque sua revisão vigente é posterior a 2020. Confirme sempre a versão, os schemas e o cronograma de cada NT.
